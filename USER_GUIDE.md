# SRT Tap Timer v1.63.1 User Guide

SRT Tap Timer は、音源を再生しながらキーボード操作で歌詞字幕のタイミングを作成・修正し、SRTファイルとして保存できる、ブラウザ上で動作するローカルSRT字幕エディターです。

主な用途は、ボーカル曲の歌詞字幕作成、AIで生成したSRTのタイミング修正、既存SRTの開始・終了時刻の再調整、静止画・音源・字幕を組み合わせた簡易字幕付きWebM動画の作成です。音源、歌詞、SRT、画像はいずれも利用者のブラウザ内で処理され、SRT Tap Timer自体がファイルをサーバーへアップロードすることはありません。

This tool is a browser-based local SRT subtitle timing editor. It is designed for creating and re-timing lyric subtitles while listening to audio, and for creating a simple subtitle video from an image, audio, and completed subtitle rows.

## このツールでできること

このバージョンでは、次の作業ができます。

- メタタグ付き歌詞テキストから、歌詞行だけを抽出する
- 既存SRTを読み込んで、開始時刻・終了時刻・字幕本文を再編集する
- 音源を再生しながら、`R` キーで字幕開始時刻を打つ
- `E` キーで字幕終了時刻を打つ
- 新規字幕作成と既存SRT再編集で、Eキーの挙動を切り替える
- `Z` キーで直前の編集操作をアンドゥする
- 行の追加・削除を行う
- 開始時刻、終了時刻、または両方のタイムスタンプを消去する
- チェックした複数行をまとめて選択・解除する
- 選択行を任意のミリ秒だけ一括シフトする
- 複数行字幕を `\n` マーカーで編集する
- タイムスタンプ未完成の状態でも途中保存SRTとして保存する
- 途中保存したSRTを再読み込みして作業を再開する
- 現在再生位置で表示中の字幕行を `＊` で確認する
- 字幕の重複、未設定時刻、不正タイムスタンプを画面上で確認する
- 音源の周辺波形を表示し、歌い出し・ブレイク・間奏明けの目安を確認する
- 直接編集パネルで、現在行の開始時刻・終了時刻・歌詞本文を手入力で修正する
- 静止画、音源、完成字幕行から、簡易字幕付きWebM動画を作成する
- 横長動画（16:9 / 1280×720）とショート動画（9:16 / 720×1280）を切り替える
- 同梱BATで、WebMをMP4へ、MP3をWAVへ変換する
- 作業ファイルJSONとして、字幕行・編集状態・主要設定を保存し、後で復元する
- タップ入力オフセット補正モードで、クリック音に合わせたキー入力から通常オフセットの推奨値を測定する
- `W` で、補正オフセット付きのR操作を行う
- `Shift + R` で、現在行の開始時刻を設定しながら、1つ前の行の終了時刻も同じ時刻へ強制的に合わせる
- `Shift + W` で、補正オフセット付きのShift+R操作を行う
- `Shift + E` で、E再編集モード設定に関係なく、最後にRでONした行の終了時刻を設定する
- `Shift + ←` / `Shift + →` で、現在行の開始時刻を0.1秒単位で微調整する

## プライバシーとローカル処理

このツールは、すべてブラウザ内で動作します。

音源ファイル、歌詞ファイル、SRTファイル、画像ファイルは、利用者のブラウザでローカルに読み込まれます。このツール自体は、音源、歌詞、字幕、画像、生成動画をサーバーへアップロードしません。

GitHub Pagesで公開した場合でも、GitHubから配信されるのは `index.html`、ドキュメント、BATファイルなどの静的ファイルだけです。利用者が選択した音源やSRTファイルは、利用者のブラウザ内で処理されます。

注意点として、このツールはブラウザの機能を使ってローカルファイルを読み込み、SRTやWebMをダウンロード保存します。元のローカルファイルを直接上書きすることはありません。

## 推奨環境と注意点

推奨環境は、Windows上の最新の Google Chrome または Microsoft Edge です。

SRT編集機能は、通常のモダンブラウザで動作する可能性があります。ただし、動画作成機能は、Canvas、MediaRecorder、HTMLMediaElement.captureStream などのブラウザ機能に依存します。特に音声付きWebM録画はブラウザ差が大きいため、ChromeまたはEdgeでの使用を推奨します。

iPhone / iPad / Safari / Firefox では、動画作成機能が動作しない、音声が入らない、WebMを保存できない、または生成された動画が期待どおりにならない可能性があります。

WebM動画はYouTubeにはアップロードできる場合がありますが、TikTokやInstagramでは受け付けられないことがあります。その場合は、同梱の `convert_webm_to_mp4.bat` を使ってMP4へ変換してください。

## 基本ワークフロー

大きく分けて、4つの使い方があります。

### 1. 歌詞テキストから新規にSRTを作る

1. 歌詞テキストを読み込む、または貼り付ける
2. **歌詞行を抽出** を押す
3. 不要行を削除・修正する
4. **字幕行リストに反映** を押す
5. 音源を読み込む
6. 音源を再生しながら、各歌詞の開始タイミングで `R` を押す
7. 必要に応じて `E` で終了時刻を設定する
8. 途中ならそのまま途中保存SRTとして保存する
9. 完成したら通常SRTとして保存する

### 2. 既存SRTを再編集する

1. SRTファイルを読み込む
2. 音源を読み込む
3. 行をダブルクリックして該当位置へ移動する
4. 開始時刻を直す場合は `R`
5. 終了時刻を直す場合は、E再編集モードで `E`
6. 必要に応じて、直接編集、行追加・削除、タイムスタンプ消去、一括シフトを行う
7. 保存する

### 3. 途中保存SRTから作業を再開する

1. `--:--,---` を含む途中保存SRTを読み込む
2. 未設定時刻は空欄として復元される
3. 音源を読み込み、未完成行のタイミングを設定する
4. 再度途中保存、または完成SRTとして保存する

### 4. 字幕付き動画を作る

1. 音源を読み込む
2. 開始・終了時刻が入った字幕行を用意する
3. **プレビュー / 動画作成** を開く
4. 動画用画像を選択する
5. 横長 / ショート動画、画像合わせ、字幕位置、文字サイズ、縁取り、黒帯を設定する
6. 動画作成を押す
7. 確認ダイアログで録画中にタブを切り替えないことを確認し、OKする
8. 録画完了後、WebMを保存する
9. 必要に応じて同梱BATでMP4へ変換する

## 画面構成

### ヘッダー

画面上部には、ツール名、言語設定、上級者モード、オールクリアがあります。

言語は日本語 / English を切り替えられます。主要なUI、動画作成ウィンドウ、録画関連メッセージ、素材ステータス表示は多言語化されています。ただし、内部ログや一部の補助的な警告文には日本語中心の表現が残る場合があります。

**上級者モード** は、細かな調整項目や危険度の高い編集操作を表示するためのモードです。これは表示だけの切り替えであり、上級者モードで設定した値は、OFFにして画面上から隠しても維持されます。

### 左側ペイン

左側には、読み込み、抽出、音源、再生、現在行の情報、直接編集、SRT保存があります。

主な要素は次の通りです。

- 読込文字コード
- メタタグ付き歌詞テキスト / SRT下書きファイル読み込み
- サンプル
- 歌詞行を抽出
- テキストクリア
- 抽出・編集用テキスト
- 字幕行リストに反映
- 音源ファイル読み込み
- 周辺波形
- 再生 / 停止、R、E、Zの操作ボタン
- 現在時刻表示
- 現在行の前後プレビュー
- Eキー挙動の設定
- タップ入力オフセット
- 波形表示オフセット
- SRT出力ファイル名
- SRT保存
- 選択行の編集

### 右側ペイン

右側には、字幕行リストと行操作ボタンがあります。

主なボタンは次の通りです。

- プレビュー / 動画作成
- 上に行追加
- 下に行追加
- 現在行を削除
- 開始消去
- 終了消去
- 時刻消去
- 現在行以降を選択
- 選択解除
- 選択行をシフト
- 字幕位置マーク更新周期

字幕行リストの主な列は次の通りです。

- 選択チェックボックス
- 表示中字幕マーク `＊`
- 行番号
- 開始時刻
- 終了時刻
- 歌詞本文

行番号は固定IDではありません。現在の行順から自動で表示され、SRT保存時にも1から順に再採番されます。

## 読み込めるファイル

### 歌詞テキスト

通常のテキストファイルを読み込めます。

歌詞抽出では、次のような行を除外します。

- 空行
- `[Verse]`、`[Chorus]`、`[Bridge]` など、角括弧だけのメタタグ行
- SRT番号行
- SRT時刻行
- `WEBVTT` 行

上級者モードでは、**半角カッコ半角文字行も削除** をONにできます。これは、英数字だけの半角カッコ行など、歌詞ではなく注釈として混ざりやすい行を除外するための補助機能です。

抽出された歌詞は、右側の字幕行リストへ反映する前に編集できます。

### SRTファイル

既存SRTを直接読み込めます。

テキストファイルまたはSRTファイルの読み込みでは、読込文字コードを指定できます。初期値は **自動** です。自動の場合、BOM確認、UTF-8厳密デコード、Shift-JISフォールバックの順で判定します。文字化けした場合は、UTF-8またはShift-JISを手動指定して読み直してください。

対応する時刻形式は次の通りです。

```text
HH:MM:SS,mmm
HH:MM:SS.mmm
MM:SS,mmm
MM:SS.mmm
--:--,---
```

`MM:SS,mmm` 形式は、内部的に `00:MM:SS,mmm` として扱います。

`--:--,---` は、このツール独自の未設定時刻マーカーです。通常のSRT仕様ではありません。

SRTとして読み込めた場合、右ペインへ字幕行が展開され、抽出・編集用テキスト欄には字幕本文のみが表示されます。SRT本文そのものがそのまま左テキスト欄に残る状態にはしません。

現在のバージョンでは、SRT読み込み直後に右ペインのスクロール位置を先頭へ戻し、1行目が見える状態で作業を開始できるようにしています。

### 音源・メディアファイル

WAV、MP3など、ブラウザが再生できる音声ファイルを読み込めます。

ファイル入力欄には次の指定をしています。

```text
accept=".wav,.mp3,audio/*"
```

そのため、通常は音声ファイルが表示されます。Chrome/Windowsでは、ファイル選択ダイアログで **すべてのファイル** を選ぶことで、MP4などのメディアファイルも選択できます。ただし、再生できるかどうかはブラウザの対応に依存します。

Sunoの無料ユーザーがMP3しかダウンロードできない場合や、波形解析・再生互換性を高めたい場合は、同梱の `convert_mp3_to_wav.bat` でWAVへ変換できます。

### 動画作成用画像

動画作成ウィンドウでは、静止画ファイルを読み込めます。ブラウザが画像として読み込めるPNG、JPEG、WebPなどが対象です。

画像は動画作成時に必須です。画像を読み込んでいない場合、動画作成ボタンは無効になります。画像なしでも動画作成ウィンドウのプレビュー領域は表示できますが、実際の動画作成には画像が必要です。

## ファイル名と出力ファイル名の扱い

### SRT出力ファイル名

SRT出力ファイル名は、基本的には歌詞/SRT側ファイル名が優先されます。

歌詞テキストまたはSRTを読み込むと、そのファイル名をもとにSRT出力ファイル名が設定されます。たとえば、`sample.txt` や `sample.srt` を読み込むと、出力ファイル名欄は `sample.srt` になります。

音源ファイル読み込み時にも、歌詞/SRTファイルがまだ選択されていない場合に限り、音源ファイル名をもとにSRT出力ファイル名を設定します。

優先順位は次の通りです。

```text
SRT出力名:
歌詞/SRTファイル名 ＞ 音源ファイル名 ＞ subtitles.srt
```

ユーザーが出力ファイル名欄を手入力で変更した場合は、自動上書きされません。

### 動画出力ファイル名

動画作成で保存されるWebMファイル名は、基本的には音源ファイル名をもとにします。

たとえば、音源が次の場合、

```text
sample_song.mp3
```

動画ファイル名は次のようになります。

```text
sample_song_subtitled.webm
```

優先順位は次の通りです。

```text
動画出力名:
音源ファイル名 ＞ SRT出力名の基準名 ＞ audio_subtitled.webm
```

### ファイル選択ダイアログをキャンセルした場合

一度ファイルを読み込んだ後に、もう一度 **ファイルを選択** を押してダイアログを開き、そこでキャンセルすると、ブラウザのfile input表示は **選択されていません** に戻ることがあります。

この場合でも、すでに読み込まれた字幕データ、音源、画像がアプリ内部で保持されていれば、そのまま作業を続けられます。これは、同じファイルを再選択して再読み込みできるようにするためのブラウザ側表示とのズレです。内部状態が失われたとは限りません。

## 作業保存・作業復元

作業保存・作業復元では、SRTとは別に、SRT Tap Timer専用の作業ファイルJSONを保存・復元できます。

これは、通常の字幕プレイヤーで使うSRTではなく、SRT Tap Timerで作業を中断し、後で再開するための保存形式です。ファイル名は次のような形式になります。

```text
曲名.srt-tap-project.json
```

### 保存されるもの

作業保存では、主に次の情報を保存します。

```text
字幕行データ
- 行順
- 歌詞本文
- 開始時刻
- 終了時刻
- 選択チェック状態

作業位置
- 現在行
- Rでタイミング中の行
- 保存時の再生位置メモ

左側テキスト
- 抽出・編集用テキスト欄の内容

SRT出力設定
- SRT出力ファイル名
- 内部的な出力名基準

編集設定
- 言語
- 上級者モード
- 読込文字コード
- E再編集モード
- Rで前行終了を自動詰めする設定
- タップ入力オフセット
- 波形表示オフセット
- 字幕位置マーク更新周期
- 波形表示ON/OFF

動画作成設定
- ショート動画ON/OFF
- 画像合わせ
- 字幕の高さ
- 文字サイズ
- 文字に縁取り
- 背景に黒帯

元ファイル名メモ
- テキスト／SRTファイル名
- 音源ファイル名
- 画像ファイル名
```

### 保存されないもの

作業保存では、次のものは保存しません。

```text
音源ファイル本体
画像ファイル本体
生成済みWebM本体
Undo履歴
波形解析済みデータ
Object URL
file input の状態
MediaRecorderの途中状態
録画中の中間データ
```

音源ファイルや画像ファイルそのものをJSONへ埋め込むと、ファイルサイズが非常に大きくなり、ブラウザ上の単体HTMLアプリとして扱いにくくなります。そのため、作業復元後は、必要に応じて音源ファイルと画像ファイルを再選択してください。

### 作業保存

画面上部の **作業保存** を押すと、現在の作業状態をJSONファイルとしてダウンロード保存します。

この保存は、SRT保存とは別です。字幕プレイヤーで使うファイルが必要な場合は、従来どおり **SRT保存** を使ってください。

### 作業復元

画面上部の **作業復元** を押すと、作業ファイルJSONを選択できます。

この時点では、現在の作業はまだ消えません。実際に作業ファイルを選択して、ファイル選択ダイアログでOKした後、JSONの読み込みと解析に成功した場合にだけ、現在の作業が復元内容で置き換えられます。

ファイル選択ダイアログでキャンセルした場合、または作業ファイルとして読み込めない場合、現在の作業は維持されます。

復元後には、次の情報をダイアログで表示します。

```text
ファイル本体は読み込まれていません。
テキスト／SRTファイル名
音源ファイル名
画像ファイル名
```

これは、復元されたのが作業状態であり、元の音源や画像ファイルそのものではないことを明示するためです。



## 歌詞テキストから字幕行を作る

歌詞テキストを読み込む、または貼り付けた後、**歌詞行を抽出** を押すと、メタタグなどを除外した歌詞行が左側の編集欄に残ります。

必要に応じてこの欄を編集してから、**字幕行リストに反映** を押します。これにより、右側の字幕行リストに1行ずつ展開されます。

この時点では、各行の開始時刻・終了時刻は未設定です。

歌詞テキストを読み込んだ場合、E再編集モードは自動的にOFFになります。これは、歌詞テキストから新規字幕作成を行う場合、通常は `R → 次のRで前行を閉じる` という流れの方が自然だからです。

## 上級者モード

初期状態は、基本操作を中心にした表示です。画面上部ヘッダーの **上級者モード** をONにすると、細かい設定や編集ボタンが表示されます。

上級者モードで表示される主な項目は次の通りです。

```text
歌詞抽出時に半角カッコ半角文字行も削除するオプション
Rで開始時刻をセットしたときの前行終了自動詰め
Eの作用先切替（SRT再編集向け）
上に行追加
開始消去 / 終了消去
選択行を任意msシフト
字幕位置マーク更新周期
タップ入力オフセット
波形表示オフセット
波形表示ON/OFF
```

初心者モードでも、基本的な読み込み、歌詞抽出、字幕行反映、再生、R/E/Z、下に行追加、現在行削除、時刻消去、プレビュー / 動画作成、SRT保存は使えます。

## 周辺波形表示

音源ファイルを読み込むと、音源コントロールの下に周辺波形が表示されます。

この波形は、音源の音量エンベロープを簡易的に可視化したものです。ボーカル検出やビート検出ではありません。イントロ明け、間奏明け、ブレイク明けなど、音の立ち上がりを視覚的に把握しやすくするための補助表示です。

波形上には、現在再生位置を示す反転表示の縦線が固定表示されます。表示範囲は現在位置の過去2秒から未来8秒で、再生に合わせて波形側がスクロールします。波形クリックによるシークは行いません。シークはメディアプレイヤーまたは左右キーで行います。

波形解析にはブラウザの Web Audio API を使います。ファイル形式やブラウザの対応状況によっては、音源自体は再生できても波形解析に失敗することがあります。その場合も、通常の再生や字幕編集は継続できます。

上級者モードでは、**波形表示を有効にする** チェックボックスで波形解析と表示をON/OFFできます。OFFにした状態で音源を読み込んだ場合、Web Audio APIによる音源デコードは行いません。読み込み済みの波形もOFF時に破棄します。

波形欄に `waveform.ready` が表示される場合があります。これは波形解析が完了した状態を示す内部メッセージです。編集や保存に影響はありません。

## タップ入力オフセット

上級者モードでは、右側ツールバーの **タップ入力オフセット** を設定できます。

これは、`R` または `E` を押した時刻に、指定したミリ秒を機械的に加算または減算して記録する機能です。

例：

```text
タップ入力オフセット: -120ms
実際にRを押した時刻: 00:01:23,500
記録される開始時刻: 00:01:23,380
```

対象になるのは、`R` と `E` によって新しく記録される時刻だけです。

対象になるもの：

```text
Rで設定する開始時刻
Rで前のタイミング中行を閉じる終了時刻
Eで設定する終了時刻
```

対象にならないもの：

```text
既存SRTの読み込み時刻
一括シフト
手動編集済みの時刻
最後の字幕終了時刻の保存時自動補完
波形表示
メディアプレイヤーの再生位置
動画作成時の字幕表示タイミング
```

初期値は0msです。負数を入れると、実際に押した時刻より前に記録されます。正数を入れると、実際に押した時刻より後に記録されます。


### タップ入力オフセット補正モード

上級者モードでは、**タップ入力オフセット** の横にある **補正** ボタンから、タップ入力オフセット補正モードを開けます。

この補正モードは、通常の `R` / `E` 入力に使う基礎オフセットを、クリック音に合わせた `R` 入力から測定するための補助機能です。W補正オフセットは測定対象ではありません。

v1.63では、補正モードの設計を変更し、画面上のR/Eターゲット表示を使わない方式にしています。タイミング基準は画面表示ではなく、クリック音です。

補正モードを開くと、内部生成されたクリックトラックがHTMLAudioElementで再生されます。

```text
0.5秒ごとに低いクリック音
4回に1回だけ高いクリック音
高いクリック音は2秒ごと
```

ユーザーは、高いクリック音に合わせて `R` キーを押します。低いクリック音は、リズムをつかむためのガイドです。画面を見るのではなく、音を基準にして押してください。

補正モードでも、通常作業と同じくHTMLAudioElementの `currentTime` を使います。これにより、通常の字幕作成時の構造に近づけています。

```text
通常作業:
音源HTMLAudioElementを聞く
→ R/Eを押す
→ audio.currentTime を字幕時刻として使う

補正モード:
クリックトラックHTMLAudioElementを聞く
→ Rを押す
→ calibrationAudio.currentTime と高クリック予定時刻の差を測る
```

オフセット計算は次の考え方です。

```text
高クリックより 400ms 早くRを押した → 推奨補正は +400ms
高クリックより 300ms 遅くRを押した → 推奨補正は -300ms
```

高いクリック音の有効範囲は、予定時刻の前後 `800ms` です。高クリックは2秒ごとなので、この範囲なら隣の高クリックと混同しにくく、早押し側と遅押し側の両方を拾えます。

推奨オフセットは中央値で計算します。ただし、偶数回では中央値を更新せず、奇数回に到達したときに更新します。

```text
1回目 → 1回の中央値
2回目 → 1回目の値を維持
3回目 → 直近3回の中央値
4回目 → 3回目の値を維持
5回目 → 直近5回の中央値
6回目 → 5回目の値を維持
7回目 → 直近7回の中央値
8回目 → 7回目の値を維持
9回目 → 直近9回の中央値
10回目以降 → 直近9回の中央値を毎回更新
```

測定を止める場合は **終了** を押します。終了後は推奨オフセットが固定表示されます。この値を使う場合は **補正** を押します。使わない場合は **キャンセル** を押します。

**補正** を押した場合だけ、メイン画面のタップ入力オフセット欄に推奨値が反映されます。キャンセルした場合、現在のタップ入力オフセットは変更されません。

この機能は、音声出力環境、キーボード、ユーザーの反応速度を含めた実用上の基礎補正を決めるためのものです。Bluetoothイヤホン、USBオーディオ、スピーカーなど、音声出力環境を変えた場合は、測定値も変わる可能性があります。

## 波形表示オフセット

上級者モードでは、右ペイン上部の **波形表示オフセット** を設定できます。

これは周辺波形表示の基準時刻だけを補正する機能です。R/Eで記録する字幕時刻、動画作成、メディアプレイヤーの再生位置には影響しません。

内部的には次のように扱います。

```text
波形表示上の基準時刻 = audio.currentTime + 波形表示オフセット
```

たとえば、PC側のオーディオインターフェイスやBluetoothなどにより、実際に耳に届く音が0.5秒遅れている場合は、タップ入力オフセットと波形表示オフセットの両方を `-500ms` にする、という使い方になります。

## 既存SRTを読み込んで再編集する

既存SRTを読み込むと、右側の字幕行リストに本文、開始時刻、終了時刻が展開されます。

読み込んだSRTのすべての字幕行に開始時刻・終了時刻の両方が入っている場合、E再編集モードは自動的にONになります。

この判定では、`終了 > 開始` であるかどうかは見ません。たとえば、終了時刻が開始時刻より前になっている不正な行があっても、開始・終了の両方が入っていれば、完成済みSRTの再編集とみなします。このような不正タイミングこそ、再編集で直す対象だからです。

一方、`--:--,---` を含む途中保存SRTを読み込んだ場合は、E再編集モードはOFFになります。これは、新規字幕作成の続きである可能性が高いためです。

## 途中保存SRTの仕様

通常のSRTでは、各字幕ブロックに開始時刻と終了時刻が必要です。しかし、作業中には、まだ時刻が入っていない行や、開始だけ入っている行が存在します。

SRT Tap Timer では、途中保存のために、未設定時刻を次のマーカーで保存します。

```text
--:--,---
```

例：

```srt
1
00:00:12,340 --> 00:00:15,100
完成した行

2
00:00:15,100 --> --:--,---
開始だけ設定した行

3
--:--,--- --> --:--,---
まだ時刻がない行
```

このようなファイルは、通常の字幕プレイヤーでは正しく読めない可能性があります。しかし、SRT Tap Timerでは再読み込みでき、未設定時刻は編集画面上の空欄として復元されます。

未設定または不正なタイムスタンプがある状態でSRT保存しようとすると、確認ダイアログが表示されます。

途中保存では、すべての字幕行が保存されます。タイムスタンプが未完成だからといって、行が削除されることはありません。

## タイミング入力の基本操作

音源を読み込んだ後、キーボードで字幕タイミングを設定します。

### Rキー

`R` は、現在行の開始時刻を現在の音源位置に設定します。

通常の新規字幕作成では、ある行で `R` を押し、次の行で `R` を押すと、前の行の終了時刻が次の `R` の時刻で自動的に設定されます。

`R` を押した後、現在行は次の行へ進みます。

### Eキー

`E` は、字幕の終了時刻を設定します。

ただし、Eキーの対象は、Eモード設定によって変わります。詳細は次の章を参照してください。

### 矢印キー

`↑` / `↓` で現在行を移動します。

`←` / `→` で音源を2秒戻す／進めます。

### 自動先読みスクロール

R/Eでリアルタイムに字幕行を進めるとき、現在行が右ペインの下側まで進んだ場合は、現在行が少し上の読みやすい位置に来るように自動でスクロールします。これにより、入力中に次以降の歌詞行を見失いにくくなります。

現在のバージョンでは、既存SRTの再編集で `↓` キーにより現在行を下へ移動する場合も、同じ考え方で下方向オートスクロールします。タイミングがすでに入っているSRTを聴きながらカーソル移動中心で再編集する場合に、現在行が右ペイン下端へ詰まりにくくなります。

通常の行クリックや上方向移動では、従来どおり最小限のスクロールに留めます。

---

## Eキーの2つのモード

Eキーには2つのモードがあります。

### 1. 新規作成モード

E再編集モードがOFFの場合、`E` は「最後にRで開始した行」の終了時刻を設定します。

これは、歌詞テキストから新規に字幕を作る場合に向いています。

例：

1. 行10の歌い出しで `R`
2. 選択行は行11へ進む
3. 行10の歌い終わりで `E`
4. 行10の終了時刻が設定される

このモードでは、Eの対象行が茶色系の背景で表示されます。

### 2. SRT再編集モード

E再編集モードがONの場合、`E` は「現在選択中の行」の終了時刻を設定し、その後、次の行へ進みます。

これは、既存SRTの終了時刻を連続して直す場合に向いています。

例：

1. 行10を選択
2. 歌い終わりで `E`
3. 行10の終了時刻が設定される
4. 自動で行11へ進む
5. 行11の歌い終わりで `E`

既存SRTを読み込んだ場合、全行に開始・終了時刻が入っていれば、このモードは自動ONになります。

### Shift + E

`Shift + E` は、E再編集モードのON/OFFに関係なく、常に新規作成モード側のOFF動作を行います。つまり、「最後にRでONした行」の終了時刻を現在時刻に設定します。

E再編集モードをONにして既存SRTを直している途中でも、一時的に「最後にRで開始した行を閉じたい」場合に使います。`Shift + E` は、現在選択中の行の終了時刻を変更するのではなく、activeIndex、つまり最後にRでONした行を対象にします。

この操作は、すでに時間情報が入っているAI生成SRTをほぼ新規作成に近い感覚で再タップしているときに有効です。通常の `E` はSRT再編集モードに従って現在行を閉じますが、`Shift + E` はモード設定を一時的に無視して、最後に `R` で開始した行を閉じられます。

## Shift系ショートカットによる低品質SRT再編集

AI生成SRTや自動文字起こしSRTでは、字幕テキストは使える一方で、時間情報の品質が低く、実質的には新規作成に近い再タップが必要になることがあります。

このようなケースでは、既存SRTの「開始時刻・終了時刻がすでに入っている」という状態が、かえって作業の邪魔になる場合があります。通常のR/E操作だけでは、既存の終了時刻を守ろうとする安全処理や、E再編集モードの対象行切替が作業感覚と合わないことがあります。

このバージョンでは、そのような低品質SRTを高速に再構築するために、Shift系ショートカットを用意しています。

### W：補正オフセット付きのR

`W` は、通常の `R` と同じ処理を行いますが、入力される時刻にW補正オフセットを追加します。

通常の `R` は、現在の音源位置にタップ入力オフセットを加えた時刻を使います。`W` はさらにW補正オフセットを加えます。

```text
Rで入力される時刻 = audio.currentTime + タップ入力オフセット
Wで入力される時刻 = audio.currentTime + タップ入力オフセット + W補正オフセット
```

W補正オフセットを `-200ms` にしておくと、実際にキーを押した瞬間より0.2秒早い時刻が入力されます。歌い出しを認識してからキーを押すまでの遅れを、必要な箇所だけ補正するための操作です。

`W` の後続処理は通常の `R` と同じです。前回 `R` または `W` で開始した行を閉じる、現在行の開始時刻を設定する、必要に応じて現在行を次へ進める、という流れは変わりません。

### Shift + W：補正オフセット付きのShift+R

`Shift + W` は、`Shift + R` と同じ境界修正を、W補正オフセットを加えた時刻で行います。

```text
現在行の開始時刻 = 補正後の現在時刻
1つ前の行の終了時刻 = 補正後の現在時刻
現在行の終了時刻 = 変更しない
activeIndex = 使わない
```

既存SRTの境界修正をしながら、押鍵タイミングの反応遅れも補正したい場合に使います。

### Shift + R：現在行開始と前行終了を同時に合わせる

`Shift + R` は、完成済みSRT再編集用の境界修正操作です。

現在行の開始時刻を現在時刻に設定し、1つ前の行の終了時刻も同じ時刻へ強制的に合わせます。通常の `R` とは違い、現在行の既存終了時刻は自動で空欄にしません。また、通常Rの activeIndex による「前回Rで開始した行を閉じる」処理も使いません。

内部的には、次のような操作です。

```text
現在行の開始時刻 = 現在時刻
1つ前の行の終了時刻 = 現在時刻
現在行の終了時刻 = 変更しない
activeIndex = 使わない
```

これは、既存SRTを聴きながら「次の字幕の開始＝前の字幕の終了」に詰め直したい場合に向いています。たとえば、行10の終わりと行11の入りの境界がずれている場合、行11を選択して正しい入りで `Shift + R` を押すと、行11の開始時刻と行10の終了時刻を同時に修正できます。

### Shift + E：Eモードに関係なく最後にRでONした行を閉じる

`Shift + E` は、E再編集モードのON/OFFに関係なく、常に「最後にRでONした行」を閉じます。

E再編集モードONで既存SRTを修正していると、通常の `E` は現在選択中の行の終了時刻を変更します。しかし、低品質SRTをほぼ新規作成として再タップしているときには、現在行ではなく、最後に `R` で開始した行を閉じたい場面があります。そのときは `Shift + E` を使います。

### Shift + ← / Shift + →：現在行開始を0.1秒単位で微調整する

現在行の開始時刻を0.1秒単位で微調整できます。

```text
Shift + ← : 現在行の開始時刻を0.1秒早める
Shift + → : 現在行の開始時刻を0.1秒遅くする
```

`Shift + ←` で現在行の開始時刻を早めた結果、1つ前の字幕の終了時刻と重複した場合は、1つ前の字幕の終了時刻を現在行の新しい開始時刻まで詰めます。これにより、前行との重複を避けながら、現在行の入りだけを早められます。

`Shift + →` で現在行の開始時刻を遅くする場合は、遅くする前の現在行開始時刻と、1つ前の字幕の終了時刻が同じだった場合に限り、1つ前の字幕の終了時刻も0.1秒遅くします。すでに前行との間にギャップがある場合は、前行の終了時刻は変更しません。

この微調整は、AI生成SRTや既存SRTを実運用で細かく追い込むための操作です。大きくずれている行は `Shift + R` で境界ごと修正し、少しだけ早い・遅い行は `Shift + ←` / `Shift + →` で微調整する、という使い分けができます。

### 低品質SRTを再タップする場合の作業例

時間情報が入っているものの品質が低いSRTでは、次のような運用が現実的です。

```text
1. SRTを読み込む
2. 音源を読み込む
3. ずれている行へカーソル移動する
4. 次の字幕の正しい入りで Shift + R を押し、現在行開始と前行終了を同時に修正する
   - 反応遅れを補正したい場合は Shift + W を使う
5. 必要に応じて Shift + E で、最後にRで開始した行を閉じる
6. 微妙な入りのズレは Shift + ← / Shift + → で0.1秒単位で調整する
7. 仕上げに通常の E や直接編集で終了時刻を整える
```

この運用では、既存SRTのテキスト行は利用しつつ、タイミングについてはほぼ新規作成に近い感覚で作り直せます。


### v1.63.1 タップ時刻取得順序の改善

v1.63.1では、`R` / `W` / `Shift + R` / `Shift + W` および `E` / `Shift + E` で字幕時刻を記録する際、まず補正後の現在時刻を取得し、その後でUndo履歴保存や字幕行更新を行うようにしました。

これにより、字幕行数が多い場合でも、Undo履歴保存のための内部処理が `audio.currentTime` の取得を遅らせにくくなります。操作仕様や保存される字幕内容は変わりません。


## Rキーの打ち直し仕様

作業中に、`R` を押すタイミングが少し遅かったと気づくことがあります。

その場合は、次のように操作できます。

1. `↑` で1つ前の行に戻る
2. `←` で音源を少し戻す
3. 正しい位置で `R` を押し直す

このとき、同じ行を `R` で打ち直しても、その行の終了時刻を同じ時刻で閉じることはありません。つまり、開始時刻と終了時刻が同一になるゼロ長字幕が残らないようになっています。

また、`R` で開始時刻を打ち直した結果、既存の終了時刻が開始時刻以下になった場合は、終了時刻を未設定に戻します。終了時刻は、次の `R` または `E` で自然に再設定できます。

### Rで前行終了を自動詰めする設定

上級者モードでは、**Rで開始時刻をセットしたとき、前行の終了時刻が食い込んでいれば現在行の開始時刻まで詰める** 設定があります。

この設定がONの場合、現在行の開始時刻をRで打ったとき、直前行の終了時刻が現在行の開始時刻より後に食い込んでいれば、直前行の終了時刻を現在行の開始時刻まで詰めます。

既存SRTの修正時に、字幕の重なりを減らすための補助機能です。

## 行の追加・削除

右側の字幕編集画面上部に、次のボタンがあります。

| ボタン | 動作 |
|---|---|
| 上に行追加 | 現在行の上に空行を追加 |
| 下に行追加 | 現在行の下に空行を追加 |
| 現在行を削除 | 現在行を削除 |

キーボードショートカットもあります。

| キー | 動作 |
|---|---|
| `Insert` | 現在行の下に空行を追加 |
| `Shift + Insert` | 現在行の上に空行を追加 |
| `Delete` | 現在行を削除 |

追加された行は、歌詞本文が空で、開始時刻・終了時刻も未設定です。

行追加・行削除を行うと、字幕番号は自動で再採番されます。行番号は固定IDではなく、現在の行順から計算されます。

行追加・行削除は `Z` でアンドゥできます。

テキスト入力欄、数値入力欄、セレクトボックス、編集可能な歌詞セルにフォーカスがある場合、`Insert`、`Delete` などのショートカットは通常の入力操作を優先し、行操作としては動作しません。

## タイムスタンプ消去

右側の字幕編集画面上部に、次のボタンがあります。

| ボタン | 動作 |
|---|---|
| 開始消去 | 開始時刻だけを消去 |
| 終了消去 | 終了時刻だけを消去 |
| 時刻消去 | 開始時刻と終了時刻の両方を消去 |

チェックボックスで選択された行がある場合は、その複数行が対象になります。チェックされた行がない場合は、現在行が対象になります。

`Backspace` は **時刻消去** と同じ操作です。

これらの操作は `Z` でアンドゥできます。

テキスト入力欄、数値入力欄、セレクトボックス、編集可能な歌詞セルにフォーカスがある場合、`Backspace` は通常の文字削除として扱われ、タイムスタンプ消去にはなりません。

## 複数行選択と一括シフト

字幕行リストの左端には選択チェックボックスがあります。

チェックした行がある場合、開始消去、終了消去、時刻消去、一括シフトなどの対象は、チェックされた複数行になります。チェックした行がない場合は、現在行が対象になります。

### 現在行以降を選択

**現在行以降を選択** を押すと、現在行から最後の行までをまとめて選択します。

これは、ある地点以降の字幕タイミングを一括で前後にずらしたい場合に使います。

### 選択解除

**選択解除** を押すと、チェックされた行の選択をすべて解除します。

### 選択行をシフト

上級者モードでは、選択行を任意のミリ秒だけ一括シフトできます。

例：

```text
選択行をシフト: -250
```

この場合、対象行の開始時刻と終了時刻が250ms前に移動します。

未設定の時刻は未設定のままです。負の時刻になる場合は0未満にならないよう補正されます。

## 複数行字幕

1つの字幕ブロック内で改行したい場合は、歌詞欄に次のマーカーを入力します。

```text
\n
```

例：

```text
雨あがりの道\n靴音がそろう
```

SRT保存時には、次のように実改行として出力されます。

```srt
雨あがりの道
靴音がそろう
```

日本語環境では、半角バックスラッシュが半角円記号のように表示される場合があります。このツールでは、半角バックスラッシュ+n と、半角円記号+n をどちらも改行として扱います。

全角の `￥n` は改行として扱いません。

既存SRTに複数行字幕が含まれている場合、読み込み時には編集画面上で `\n` に変換して1行表示します。保存時には再び実改行に戻します。

動画作成時も、`\n` は実改行として字幕に反映されます。

## 選択行の直接編集

左ペイン中部に **選択行の編集** パネルがあります。

ここでは、現在選択している字幕行の開始時刻、終了時刻、歌詞を直接編集できます。編集した内容は即時反映されません。**選択行に反映** ボタンを押したときにだけ、右ペインの字幕行へ反映されます。

時刻は次の形式を基本とします。

```text
00:01:23,456
01:23,456
--:--,---
空欄
```

`--:--,---` と空欄は未設定扱いです。複数行字幕は、従来どおり `\n` で表します。

終了時刻が開始時刻以下の場合は確認ダイアログを出します。必要な修正作業を妨げないため、確認すれば反映できます。

この直接編集の反映はUndo対象です。一方で、SRTファイルやテキストファイルの読み込み自体はUndo対象から外しています。ファイル読み込みは新しい作業開始状態として扱います。

同じファイルを再選択して再読み込みできるように、ファイル選択ダイアログを開く直前にファイル入力欄を内部的にクリアします。

---

## 表示中字幕マーク

右側の字幕行リストには `＊` 列があります。

音源の現在位置が字幕の表示範囲内に入っている場合、その行に `＊` が表示されます。

表示中字幕マークの更新周期は、右ペイン上部の **字幕位置マーク更新周期(ms)** で変更できます。

初期値は50msです。

ただし、100行以上の字幕行を読み込んだ時点で、更新周期が100ms未満に設定されている場合は、自動的に100msへ変更されます。これは、長いSRTで画面更新が重くなることを避けるためです。

この更新周期は、メイン画面側の `＊` 表示、現在時刻表示、選択行表示、波形描画更新などの監視周期として使われます。動画作成ウィンドウ内のプレビューやWebM録画時の字幕描画周期には使われません。

## プレビュー / 動画作成

右ペイン上部の **プレビュー / 動画作成** ボタンを押すと、動画作成ウィンドウが開きます。

この画面は、従来の黒背景プレビューだけの画面ではなく、字幕のプレビューと簡易字幕付き動画作成を兼ねた画面です。

### 開くための条件

動画作成ウィンドウを開くには、少なくとも次の条件が必要です。

```text
本体側で音源が読み込まれている
開始時刻と終了時刻があり、終了時刻が開始時刻より後である字幕行が1行以上ある
```

動画作成ウィンドウを開くと、メイン画面側の音源再生は停止します。二重再生を避けるためです。

### 動画作成に使われる字幕行

動画作成ウィンドウでは、右ペインの字幕行リストから、次の条件を満たす行だけを使います。

```text
開始時刻がある
終了時刻がある
終了時刻が開始時刻より後である
歌詞本文が空ではない
```

未設定時刻を含む途中保存行、終了時刻が開始時刻以下の不正行、空行は動画作成対象になりません。

### 画像

動画作成ウィンドウの **画像ファイル** で、動画背景に使う画像を選択します。

**画像をクリア** を押すと、読み込み済み画像を解除します。ただし、録画中は画像をクリアできません。

画像を選択していない状態では、プレビュー領域に「画像プレビュー」と表示されますが、実際の動画作成はできません。動画作成には画像、音源、完成字幕行が必要です。

### 動画タイプ

**ショート動画** をOFFにすると、横長動画になります。

```text
16:9 / 1280×720
```

**ショート動画** をONにすると、縦長動画になります。

```text
9:16 / 720×1280
```

### 画像合わせ

**画像合わせ** スライダーは、画像の表示倍率を調整します。

左端は全体表示です。画像全体がキャンバス内に収まるように表示されます。

右へ動かすほど拡大します。右端は、中央切り抜き相当、または全体表示から1.77倍のうち大きい方まで拡大します。

値表示は、全体表示を基準に `x1.00`、`x1.77` のような倍率で表示されます。

### 字幕の高さ

**字幕の高さ** は、字幕を画面のどの高さに表示するかを指定します。

値はパーセントで表示されます。数値を大きくすると字幕は下側へ、小さくすると上側へ移動します。

### 文字サイズ

**文字サイズ** は、動画内字幕のフォントサイズです。

横長動画とショート動画では画面サイズが異なるため、同じ文字サイズでも見え方が変わります。小さいスマートフォン画面で見るショート動画では、やや大きめにする方が読みやすい場合があります。

### 文字に縁取り

**文字に縁取り** をONにすると、字幕文字に縁取りを付けます。背景が明るい画像でも文字を読みやすくするための設定です。

### 背景に黒帯

**背景に黒帯** をONにすると、字幕の背後に半透明の黒帯を表示します。

背景が複雑な画像や、明暗差が大きい画像では、黒帯を使うと字幕の可読性が安定します。

### 設定を初期値に戻す

**設定を初期値に戻す** を押すと、動画作成画面内の表示設定を初期状態に戻します。画像、音源、字幕データそのものを削除する操作ではありません。

### 動画プレビュー

右側の大きなキャンバスが、生成される動画のプレビューです。

このプレビューは動画作成機能専用です。録画開始時には自動停止します。

### 音源・字幕プレビュー

動画作成ウィンドウ内の音声コントロールで、音源と字幕のタイミングをプレビューできます。

ここでのプレビューは、動画作成対象になる完成字幕行だけを表示します。字幕がない時間帯は、何も表示されません。かつての「字幕位置の確認」のようなダミーテキストは表示しません。

### 動画作成

**動画作成** を押すと、ブラウザ内でWebM動画を作成します。

出力形式はWebM固定です。動画ファイル名は、基本的に音源ファイル名をもとに、`_subtitled.webm` を付けた名前になります。

録画前には確認ダイアログが表示されます。OKを押すと録画を開始します。キャンセルを押すと録画は開始されません。

### 中止

録画中に **中止** を押すと、録画を停止します。中止した場合、正常なWebMが作成されない場合があります。

### 保存

録画が完了すると **保存** リンクが有効になります。クリックするとWebMファイルをダウンロード保存できます。

---

## 録画中のタブ切り替え禁止

動画作成中は、ブラウザのタブを切り替えないでください。

録画中にタブをバックグラウンドへ回すと、ブラウザがCanvas描画や `requestAnimationFrame` の更新を間引くことがあります。その場合、WebMファイル自体は作成され、音声も入っているのに、字幕だけが入らない動画になることがあります。

これは録画失敗として明示的にエラーになるとは限りません。ファイルサイズも小さくなりやすく、見た目には「静止画＋音源」の動画になったように見えます。

そのため、動画作成ボタンを押した後には確認ダイアログが表示されます。録画が終わるまで、このタブを前面に表示したままにしてください。

## WebMからMP4への変換

SRT Tap Timerの動画作成機能は、ブラウザ内でWebM動画を作成します。

WebMはYouTubeにアップロードできる場合がありますが、TikTokやInstagramでは受け付けられないことがあります。その場合は、同梱の `convert_webm_to_mp4.bat` を使ってMP4へ変換してください。

### 使い方

1. ffmpegをインストールし、PATHを通す
2. `convert_webm_to_mp4.bat` にWebMファイルをドラッグ＆ドロップする
3. 入力と同じフォルダに、同じベース名のMP4が作成される

例：

```text
sample_subtitled.webm
→ sample_subtitled.mp4
```

### 上書き防止

同名のMP4がすでに存在する場合、変換は停止します。既存ファイルを上書きしません。

これは、配布ツールとして安全側に倒すための仕様です。

### 変換設定

MP4変換では、互換性重視で次の形式にします。

```text
video: H.264 / yuv420p / 30fps
audio: AAC / 192kbps
movflags: +faststart
```

## MP3からWAVへの変換

Sunoでは、無料ユーザーがMP3形式でしか音源をダウンロードできない場合があります。

SRT Tap TimerはMP3を読み込めますが、環境によってはWAVの方が扱いやすい場合があります。また、他の音声編集ソフトや動画編集ソフトへ渡す場合、WAVが必要になることがあります。

そのため、同梱の `convert_mp3_to_wav.bat` でMP3をWAVへ変換できます。

### 使い方

1. ffmpegをインストールし、PATHを通す
2. `convert_mp3_to_wav.bat` にMP3ファイルをドラッグ＆ドロップする
3. 入力と同じフォルダに、同じベース名のWAVが作成される

例：

```text
song.mp3
→ song.wav
```

### 上書き防止

同名のWAVがすでに存在する場合、変換は停止します。既存ファイルを上書きしません。

### 変換設定

WAV変換では、互換性重視で次の形式にします。

```text
44.1kHz / 16bit PCM / stereo
```

## 最後の字幕終了時刻の自動補完

新規作成ワークフローでは、ユーザーが最初から最後まで `R` だけで開始時刻を打っていくことがあります。この場合、最後の字幕行だけは、次の字幕行が存在しないため、終了時刻が自動では入りません。

SRT保存時に次の条件を満たす場合、最後の字幕の終了時刻を自動補完するか確認します。

```text
最後の行に開始時刻がある
最後の行の終了時刻だけが未設定である
それ以前の行には開始時刻・終了時刻があり、終了時刻が開始時刻より後である
他の未設定時刻や不正タイムスタンプがない
```

この条件を満たす場合、保存時に次の候補時刻を提示します。

```text
min(最後の字幕の開始時刻 + 10秒, 音源の終了時刻)
```

音源の終了時刻がブラウザから取得できない場合、または音源終了時刻が最後の字幕開始時刻以前である場合は、候補は次の時刻になります。

```text
最後の字幕の開始時刻 + 10秒
```

確認画面では、次の選択ができます。

```text
1: 候補時刻を設定して保存
2: 未設定のまま途中保存
キャンセルまたは空欄: 保存をキャンセル
```

`1` を選ぶと、最後の字幕の終了時刻を候補時刻に設定してから保存します。この自動補完は `Z` でアンドゥできます。

`2` を選ぶと、最後の終了時刻を `--:--,---` のまま、途中保存SRTとして保存します。この場合、通常の未完成タイムスタンプ確認は重ねて表示しません。

この補完は「最後の終了時刻だけが未設定」の場合に限って働きます。途中の行に未設定時刻がある場合や、他の不正タイムスタンプがある場合は、従来どおり途中保存確認になります。

## SRT保存

**SRT保存** を押すと、右側の字幕行リストの内容をSRTとしてダウンロード保存します。

保存時のファイル名は、左側のファイル名入力欄で指定できます。`.srt` が付いていない場合は、自動で追加されます。

すべての行に正常な開始時刻・終了時刻がある場合は、通常のSRTとして保存されます。

未設定時刻や不正なタイムスタンプがある場合は、確認ダイアログが表示されます。そのまま保存すると、未設定時刻は `--:--,---` として保存されます。

このツールは、元のローカルファイルを直接上書きしません。常に新しいダウンロードファイルとして保存します。

SRT Tap Timerには、複数のクリア操作があります。

### テキストクリア

### 編集画面クリア

オールクリアはリセット操作として扱ってください。`Z` で一部の字幕行データが戻る場合があっても、左側テキスト、読み込み済み音源、出力ファイル名などが完全に戻るとは限りません。

タップ入力オフセット補正モード中は、`E` と `R` は測定用キーとして扱われ、本体側の字幕編集操作には使われません。

入力欄、数値欄、セレクトボックス、歌詞セルを編集中の場合、通常の入力操作が優先されます。

動画作成ウィンドウ内では、本体側の `R`、`E`、`Delete`、`Backspace` などの編集ショートカットが誤発火しないようにしています。

## 重要な仕様と依存関係

このツールは、多くの操作が相互に依存しています。主な依存関係を整理します。

### 1. SRT読み込み時のEモード自動切替

読み込んだSRTの全行に開始・終了時刻がある場合、E再編集モードは自動ONになります。

1行でも未設定時刻 `--:--,---` がある場合、途中保存SRTとみなし、E再編集モードはOFFになります。

### 2. Eキーの挙動はモードに依存する

E再編集モードOFFでは、`E` は最後に `R` で開始した行を閉じます。

E再編集モードONでは、`E` は現在行の終了時刻を設定し、次の行へ進みます。

`Shift + E`は例外です。E再編集モードのON/OFFに関係なく、最後に`R`で開始した行を閉じるOFF動作を行います。

### 3. RキーとWキーはactiveIndexを使う

`R` または `W` を押すと、その行が「タイミング中」の行になります。次に別の行で `R` または `W` を押すと、前のタイミング中行の終了時刻が自動設定されます。`W` は時刻にW補正オフセットを加える点だけが異なります。

ただし、同じ行を `R` で打ち直した場合は、その行を同じ時刻で閉じません。

### 4. 行追加・削除はactiveIndexを調整する

タイミング中の行より前に行を追加した場合、内部的なタイミング中行番号もずれます。

タイミング中の行そのものを削除した場合、タイミング中行はリセットされます。

### 5. タイムスタンプ消去は複数行選択を優先する

タイムスタンプ消去は、チェックボックスで選択された行がある場合、その選択行を対象にします。

選択行がない場合だけ、現在行が対象になります。

### 6. SRT保存は全行保存する

タイムスタンプが未完成の行も削除されません。未設定時刻は `--:--,---` として保存されます。

### 7. 動画作成は完成行だけを使う

動画作成では、開始・終了の両方があり、かつ終了時刻が開始時刻より後の行だけを使います。

途中保存行、不正な行、空行は動画作成対象になりません。

### 8. 字幕位置マーク更新周期は動画作成には使わない

右側の `＊` 表示の更新周期は、メイン画面側の監視周期です。

動画作成ウィンドウ内のプレビューは `requestAnimationFrame` による別ループで描画されます。WebM録画時も、録画用の30fps相当の描画ループを別に持っています。

### 9. 最後の終了時刻だけ未設定の場合は保存時に補完提案する

最後の字幕行だけ終了時刻が未設定で、それ以外の行がすべて正常な場合、保存時に自動補完を提案します。

候補時刻は、最後の字幕開始から10秒後と音源終了時刻の早い方です。音源終了時刻が取得できない場合は、最後の字幕開始から10秒後を使います。

この補完は、途中に未完成行がある場合や、他の不正タイムスタンプがある場合には発動しません。

### 10. オールクリアは完全にはアンドゥできない

`Z` で字幕行データが戻る場合がありますが、左側テキスト、読み込み済み音源、出力ファイル名は完全には戻りません。

オールクリアはリセット操作として扱ってください。

### 11. 動画作成中のタブ切り替えは字幕欠落を起こす可能性がある

録画中にブラウザタブを切り替えると、WebM自体は作成されても字幕が入らないことがあります。

動画作成時は録画完了までタブを前面に表示したままにしてください。

### 12. 同梱BATは上書きしない

`convert_webm_to_mp4.bat` と `convert_mp3_to_wav.bat` は、同名の出力ファイルが存在する場合、変換せずに停止します。

既存ファイルを誤って上書きしないための仕様です。

## README / USER_GUIDE / CHANGELOG の役割

このリポジトリでは、ドキュメントの役割を分離しています。

```text
README.md      : リポジトリの入口。概要、使い方へのリンク、ライセンス、公開URLなど。
USER_GUIDE.md  : フルスペックの操作マニュアル兼仕様書。
CHANGELOG.md   : バージョンごとの変更履歴。
LICENSE        : MIT License本文。
```

README.md にすべての操作説明と更新履歴を入れると、リポジトリのトップページが長くなりすぎ、利用者にも開発者にも読みにくくなります。

そのため、README.md は短い案内ページとして維持し、詳細仕様は USER_GUIDE.md に集約します。更新履歴は CHANGELOG.md に分離します。

```text
index.html
README.md
USER_GUIDE.md
CHANGELOG.md
LICENSE
convert_webm_to_mp4.bat
convert_mp3_to_wav.bat
```

```text
Source: Deploy from a branch
Branch: main
Folder: / (root)
```

```text
https://<username>.github.io/<repository-name>/
```
